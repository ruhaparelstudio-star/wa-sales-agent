You are implementing Phase 6 of a chat-agent refactor.

## Objective
Create a unified turn outcome model so every turn ends with one explicit outcome status.

## Problem
Current runtime has multiple valid no-reply paths such as guardrail block, superseded inbound, missing WhatsApp agent, disconnected agent, empty payload, and fallback-not-dispatched cases. Operationally these are hard to distinguish.

## Scope
Implement:
1. TurnOutcome builder
2. TurnOutcomeType enum
3. no-reply reason enum or constants
4. integration in orchestrator and dispatch path
5. logging/tracing update to emit one final outcome per turn

## Minimum outcome types
- replied
- fallback_replied
- no_reply_guardrail
- no_reply_superseded
- no_reply_disconnected
- no_reply_empty_payload
- no_reply_missing_agent
- no_reply_duplicate
- no_reply_blocked
- reply_failed_dispatch

## Required behavior
- every processed turn must end with exactly one outcome
- dispatch success/failure must be reflected in outcome
- fallback-created-but-not-sent must not look like replied
- superseded must not look like silent failure
- log payload should include decision_action and no_reply_reason if relevant

## Constraints
- integrate without rewriting entire logging stack
- add adapters if older log structure must remain temporarily
- keep outcome assignment deterministic

## Tests required
1. normal replied outcome
2. fallback replied outcome
3. superseded no-reply outcome
4. disconnected no-reply outcome
5. empty payload no-reply outcome
6. dispatch failure outcome
7. duplicate no-reply outcome

## Out of scope
- media path fix itself
- full analytics dashboard

## Deliverables
1. outcome model
2. integration patches
3. tests
4. short outcome matrix documentation

## Definition of Done
- every turn has one explicit final outcome
- no-reply ambiguity is greatly reduced
