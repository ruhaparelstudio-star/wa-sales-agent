You are implementing Phase 4 of a chat-agent refactor.

## Objective
Replace hidden branch-order routing with an explicit priority-based routing table.

## Problem
Current routing logic is embedded in a long sequence of if/else branches inside the orchestrator. This makes priority implicit, fragile, and hard to test.

## Scope
Implement:
1. `RoutingRule`
2. `RoutingPriorityMap`
3. `BusinessActionRouter`
4. migration of a subset of existing actions to the new router

## Minimum actions to route explicitly
- opt_out
- handoff
- pricelist inquiry
- package details inquiry
- booking field reply

## Required behavior
- each rule has:
  - name
  - priority
  - condition callback or matcher
  - target action
- router selects the highest-priority matching rule
- router can log evaluated rules and chosen rule
- router consumes FinalTurnDecision and relevant signals

## Constraints
- do not migrate every branch in this phase
- preserve current behavior for non-migrated branches
- add fallback to old path for unmigrated actions
- make priorities obvious and documented

## Tests required
1. highest-priority rule wins
2. non-matching rules skipped
3. migrated action routes correctly
4. fallback to legacy path works
5. routing trace log contains chosen rule

## Out of scope
- full business handler migration
- media path fix
- booking persistence rewrite

## Deliverables
1. routing classes
2. subset migration
3. tests
4. routing priority documentation

## Definition of Done
- selected core actions now route through explicit priority rules
- branch priority is no longer hidden inside if ordering
