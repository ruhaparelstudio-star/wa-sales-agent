You are implementing Phase 8 of a chat-agent refactor.

## Objective
Document and partially consolidate dedupe behavior into an explicit dedupe contract.

## Problem
Inbound dedupe, processing dedupe, outbound dedupe, and sidecar dedupe exist across multiple layers. Safety is strong, but the mechanism is difficult to audit and reason about.

## Scope
Implement:
1. DedupeContract markdown/spec
2. DedupeDecision helper or DTO
3. explicit reason labels for duplicate vs superseded vs skipped
4. integration in the most critical paths only

## Required behavior
For each critical dedupe point, define:
- dedupe layer name
- key used
- TTL or lifetime
- trigger condition
- outcome when triggered
- whether it stops processing, dispatch, or both

## Constraints
- do not rip out all existing dedupe logic
- make it observable first
- only consolidate the most critical dedupe labels and reporting
- preserve behavior

## Tests required
1. inbound duplicate reason visible
2. superseded reason visible and distinct
3. outbound duplicate reason visible
4. no confusion between duplicate and disconnected send failure

## Deliverables
1. dedupe spec
2. minimal code integration
3. tests
4. summary of current dedupe layers

## Definition of Done
- duplicate handling becomes auditable
- duplicate, superseded, and send failure are distinguishable in logs/outcomes
