You are implementing Phase 3 of a chat-agent refactor.

## Objective
Create a shared conversation context contract used consistently by both the classifier and responder.

## Problem
The classifier and responder currently consume different context sets. Summary is used by the responder but not the classifier, which can lead to topic drift and inconsistent interpretation across long threads.

## Scope
Implement:
1. SharedConversationContextFactory
2. SharedConversationContext DTO
3. classifier input update to include shared context
4. responder input update to include the same shared context
5. additive integration only; avoid risky prompt rewrites

## SharedConversationContext must include
- schema_version
- conversation_id
- active_topic
- current_stage
- stage_goal
- latest_user_ask
- recent_summary
- filled_slots
- unresolved_questions
- asked_fields
- next_expected_field
- next_best_action
- memory_facts
- recent_messages

## Required behavior
- classifier must receive recent_summary
- classifier and responder must receive the same active_topic
- unresolved_questions must be available to both
- next_best_action must be included where relevant
- if summary is missing, fallback gracefully

## Constraints
- preserve current prompt structure where possible
- minimize token growth
- do not inject excessive transcript history
- do not rewrite all context assembly logic unless necessary
- keep this phase focused on context symmetry

## Tests required
1. classifier receives recent_summary
2. responder receives same active_topic as classifier
3. long-thread continuation case remains on topic
4. missing summary fallback case
5. no duplicate over-injection of history

## Out of scope
- booking validation
- routing table
- full orchestrator split

## Deliverables
1. SharedConversationContextFactory
2. DTO integration in classifier/responder path
3. tests
4. short markdown note explaining context symmetry rules

## Definition of Done
- classifier and responder now consume the same summary/topic/state contract
- context drift risk is reduced
- prompt growth remains controlled
