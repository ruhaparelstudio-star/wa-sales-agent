# DEPLOY.md — Wedding Sales Agent SaaS

## Production Deploy Guide (VPS Ubuntu 22.04)

---

## Server Requirements

```
OS:        Ubuntu 22.04 LTS
PHP:       8.4+ (extensions: BCMath, Ctype, Fileinfo, JSON, Mbstring, OpenSSL, PDO, Tokenizer, XML, Redis, GD)
Node.js:   20 LTS
MySQL:     8.0+
Redis:     7+
Nginx:     latest stable
Supervisor: latest
PM2:       latest (global npm)
Certbot:   latest
```

Install dependencies:

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y nginx mysql-server redis-server supervisor certbot python3-certbot-nginx
sudo add-apt-repository ppa:ondrej/php
sudo apt install -y php8.4-fpm php8.4-cli php8.4-bcmath php8.4-ctype php8.4-fileinfo \
    php8.4-json php8.4-mbstring php8.4-openssl php8.4-pdo php8.4-tokenizer php8.4-xml \
    php8.4-redis php8.4-gd php8.4-mysql php8.4-zip php8.4-curl
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
sudo npm install -g pm2
```

## Deploy Steps

```bash
# 1. Clone project
git clone {repo} /var/www/html
cd /var/www/html

# 2. Install PHP dependencies
composer install --no-dev --optimize-autoloader

# 3. Environment
cp .env.example .env
# Edit .env — see checklist below
nano .env

# 4. Generate app key
php artisan key:generate

# 5. Database — create DB and user first
mysql -u root -p <<'SQL'
CREATE DATABASE wedding_saas CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'wedding_user'@'localhost' IDENTIFIED BY 'strong-password';
GRANT ALL PRIVILEGES ON wedding_saas.* TO 'wedding_user'@'localhost';
FLUSH PRIVILEGES;
SQL

php artisan migrate --force

# 6. Storage
php artisan storage:link
mkdir -p storage/app/tenants
chmod -R 775 storage bootstrap/cache
chown -R www-data:www-data storage bootstrap/cache

# 7. Cache & optimize
php artisan optimize
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan event:cache

# 8. Supervisor (Horizon + scheduler)
sudo cp deployment/supervisor/wedding-saas.conf /etc/supervisor/conf.d/wedding-saas.conf
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl start all

# 9. Nginx
sudo cp deployment/nginx/production.conf /etc/nginx/sites-available/wedding-saas
sudo ln -s /etc/nginx/sites-available/wedding-saas /etc/nginx/sites-enabled/wedding-saas
sudo nginx -t && sudo systemctl reload nginx

# 10. SSL
sudo certbot --nginx -d yourdomain.com
# Certbot will auto-update the nginx config with SSL cert paths

# 11. Baileys sidecar
cd /var/www/html/baileys-svc
npm ci --production
pm2 start src/index.js --name baileys-svc
pm2 save
pm2 startup  # follow printed instructions to enable PM2 on boot

# 12. Cron (Laravel scheduler)
# Already handled by Supervisor — no crontab entry needed.
# If you prefer cron instead of supervisor scheduler:
# crontab -e
# Add: * * * * * cd /var/www/html && php artisan schedule:run >> /dev/null 2>&1
```

## Post-Deploy Verification

```bash
# Check all processes running
sudo supervisorctl status
pm2 status

# Check horizon
php artisan horizon:status

# Test queue
php artisan tinker --execute="dispatch(new \App\Modules\AgentCore\Jobs\FollowUpSchedulerJob())"

# Check logs
tail -f /var/www/html/storage/logs/laravel.log
tail -f /var/www/html/storage/logs/horizon.log

# Check Nginx
curl -I https://yourdomain.com
```

## Update / Redeploy

```bash
cd /var/www/html
git pull origin main
composer install --no-dev --optimize-autoloader

php artisan migrate --force
php artisan optimize:clear
php artisan optimize

sudo supervisorctl restart laravel-horizon
pm2 restart baileys-svc
sudo systemctl reload nginx
```

---

## Rollback

```bash
cd /var/www/html
git checkout {previous-tag}
composer install --no-dev --optimize-autoloader
php artisan migrate:rollback --step=1   # only if migration was part of update
php artisan optimize:clear && php artisan optimize
sudo supervisorctl restart laravel-horizon
```

## Horizon Dashboard Access

Horizon is protected by IP whitelist in Nginx. To access:
- Add your office IP in `deployment/nginx/production.conf` under `/horizon` block.
- Or use SSH tunnel: `ssh -L 8080:127.0.0.1:80 user@yourserver`
  then visit `http://localhost:8080/horizon`
